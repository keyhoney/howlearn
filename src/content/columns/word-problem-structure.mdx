초5 때 긴 문장제를 처음부터 끝까지 읽다가 "뭐라는 거지?" 하고 멈춘 적 있어요. 빨리 읽으려다 **한 번에 너무 많이** 붙잡으려 했거든요. 짧은 문제는 괜찮았는데, 문장이 길어지면 앞 내용이 사라지고 숫자만 떠돌았어요.

나중에 학원에서도 비슷한 패턴을 많이 봤어요. 계산을 시작하기 전에 문제만 오래 바라보다 손이 멈추는 학생이 꽤 있었거든요. 계산이 약한 경우보다, **어디가 주어진이고 어디가 구하는 것인지** 헷갈리는 자리에서 막히는 경우가 많았어요.

한 줄로 말하면, 문장제가 어렵다고 해서 항상 계산이 약한 건 아니에요. 먼저 볼 것은 **"긴 글을 어떻게 나눠 읽고 있느냐"**예요.

<MdxH2 icon="messageCircle" badge="상황 이해">
 긴 문장제에서 멈추는 이유
</MdxH2>

짧은 문제는 [문제 읽기 3단계](/columns/math-reading-three-steps)처럼 질문·숫자·조건을 빠르게 나눌 수 있어요. 그런데 문장제는 **이야기처럼 길게** 이어지거든요. 누가 무엇을 했는지, 중간에 조건이 바뀌는지, 마지막에 무엇을 구하는지가 한 덩어리로 들어옵니다.

<MdxH2 icon="brain" badge="핵심 원리">
 이론 한 줄 — 긴 지문이 작업 기억을 넘치게 할 때
</MdxH2>

문장제는 **읽기 부하**와 **수학 부하**가 겹쳐요. [**인지 부하(Cognitive Load)**](/concepts/cognitive-load)가 커지면 "버겁다"는 느낌이 계산 전에 먼저 올 수 있어요. [**작업 기억 모델(Working Memory Model)**](/concepts/working-memory-model)로 보면, "누가 했지?", "이 숫자는 뭐지?", "결국 뭘 구하지?"가 동시에 겹치면 개념을 아는 것과 별개로 시작이 막힐 수 있어요.

<MdxH2 icon="search" badge="현장 분석">
 3블록 읽기를 써 보며 알게 된 것
</MdxH2>

나는 문장제 3블록 읽기를 여러 학생과 긴 지문 문제에 적용해 봤어요. 막히는 지점이 달랐거든요.

**지문이 길면 압도됨** — 첫·마지막 문장만 읽고 **구하는 것부터** 잡게 하니 시작은 됐어요.

**숫자가 많음** — 숫자마다 **단위를 적고**, 구하는 것에 안 쓰는 숫자는 X 표시하게 하니 정리됐어요.

**숫자는 찾았는데 식이 안 됨** — 읽기는 통과한 거예요. **수식 칸**에 주어진·구하는 것만 옮기게 하니 다음 단계가 보였어요.

**응용 문장제만 막힘** — [응용 문제 구조 비교](/columns/application-problems-transfer)와 함께 보면 읽기·구조 막힘이 겹친 경우가 많았어요.

| 관찰 | 자주 보이는 패턴 | 조정 |
|------|----------------|------|
| 문제만 오래 봄 | 통째로 읽기 | 구하는 것 먼저 |
| 숫자만 떠돎 | 블록 없이 읽기 | 3블록 순서 고정 |
| 읽기 OK, 식 안 됨 | 읽기·계산 동시 | 읽기 칸 / 수식 칸 분리 |
| 빨리 읽으라 해서 더 헷갈림 | 속도만 올리기 | 구조 읽기 2회 반복 |

왜 이런 차이가 났을까요? 긴 글을 한 덩어리로 읽으려 하면 [**정보 처리(Information Processing)**](/concepts/information-processing) 변환이 늦어지고, 수식 칸으로 옮기기 전에 이미 지쳐 버릴 수 있어요.

<MdxH2 icon="target" badge="3블록">
 문장제 3블록 나누기 — 나만의 읽기 순서
</MdxH2>

나는 이제 문장제를 처음부터 끝까지 읽지 않아요. **한 번에 하나씩**만 처리해요.

**1블록: 구하는 것** — 마지막 문장(또는 "구하시오" 문장)만 먼저 읽고 밑줄을 그어요.

**2블록: 누가/무엇** — 인물·대상·사건의 주인공을 표시해요. "누가"가 없으면 **무엇**에 초점을 맞춰요.

**3블록: 주어진 숫자·관계** — 숫자에 동그라미를 치고 단위를 옆에 적어요. 관계는 화살표나 짧은 말로 표시해요.

**수학 예시 1 — 초등 사과 문제**

"민수는 사과 5개를 샀고, 2개를 먹었다. 남은 사과는 몇 개인가?"

| 블록 | 나의 표시 |
|------|----------|
| 구하는 것 | "남은 사과는 몇 개인가?" 밑줄 |
| 누가/무엇 | 민수, 사과 |
| 주어진·관계 | ⑤ 샀음, ② 먹음 → 남은 개수 |

**수학 예시 2 — 중등 속력 문장제**

"지훈이는 집에서 학교까지 시속 4km로 15분 걸렸다. 집에서 학교까지의 거리를 구하시오."

| 블록 | 나의 표시 |
|------|----------|
| 구하는 것 | "거리를 구하시오" 밑줄 |
| 누가/무엇 | 지훈, 집↔학교 |
| 주어진·관계 | 시속 4km, 시간 15분 → 거리 = 속력 × 시간 |

<MdxH2 icon="lightbulb" badge="실천 가이드">
 학생·학부모가 오늘부터 쓸 수 있는 3단계
</MdxH2>

**1단계 — 구하는 것부터 5분 읽기**
문장제를 열고 5분 안에 풀려고 하지 않아요. 마지막 문장만 읽고 구하는 것 밑줄 → 누가/무엇 표시 → 숫자·단위·관계 표시. **계산은 아직 시작하지 않아요.**

**2단계 — 읽기 칸과 수식 칸 분리**
3블록을 종이 오른쪽에 적고, 왼쪽 수식 칸에는 **주어진과 구하는 것만** 옮기세요.

**3단계 — 멈춘 블록 한 줄 기록**
오늘 **어느 블록에서 멈췄는지** 한 줄로 적어 두세요. 누가/무엇에서 막히면 읽기, 식이 안 떠오르면 풀이·공식 단계예요.

학부모 상담에서 "문장제 읽기가 너무 느려요"라는 말을 자주 들었어요. 그때 **"구하는 것·누가/무엇·숫자 중 어디에서 멈췄어?"**로 질문을 바꾸면, 속도보다 **구조**를 볼 수 있었어요.
