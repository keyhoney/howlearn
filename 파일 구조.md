# 학습과학 지식 브랜드 — 파일 구조

Next.js 16 (App Router) + MDX 기반 블로그/지식 사이트의 디렉터리 구조와 **각 요소의 역할**입니다.

---

## 루트

```
math_blog/
├── app/                    # 페이지·라우트·레이아웃 (URL별 화면 구성)
├── components/             # UI 조각 (여러 페이지에서 재사용)
├── content/                 # 글/가이드/개념 등 MDX 원본 (콘텐츠 저장소)
├── lib/                     # 데이터 조회·계산·설정 (페이지/컴포넌트가 사용)
├── public/                  # 이미지·폰트 등 정적 파일 (URL 그대로 서빙)
├── mdx-components.tsx       # MDX → React 매핑 진입점
├── next.config.ts           # Next + MDX 빌드 설정
├── package.json / tsconfig.json / tailwind.config.ts / postcss.config.mjs
└── 개발 지침.md, 세부 지침.md, README.md
```

| 경로 | 역할 |
|------|------|
| **app/** | URL과 페이지를 연결한다. 라우트별로 어떤 컴포넌트를 보여줄지, 메타데이터·JSON-LD를 어떻게 만들지 결정한다. |
| **components/** | 버튼·카드·헤더·필터 등 반복되는 UI를 컴포넌트로 묶어, app·다른 컴포넌트에서 조합해 쓴다. |
| **content/** | 블로그·가이드·개념·툴킷·전자책 등 실제 글(MDX)을 타입별 폴더에 저장한다. lib가 여기서 파일을 읽어 화면에 쓴다. |
| **lib/** | 콘텐츠 목록/상세 조회, URL 생성, 읽는 시간·관련글 계산, 사이트 정보, GA4 전송 등 **로직**을 담는다. UI는 하지 않는다. |
| **public/** | favicon, ads.txt, SVG 등 빌드 없이 그대로 `/파일명`으로 제공할 정적 자산을 둔다. |
| **mdx-components.tsx** | MDX가 렌더링할 때 사용할 React 컴포넌트를 한곳에서 지정한다. (내부적으로 lib/mdx-components 사용) |

---

## app/ (라우트·레이아웃)

**역할:** URL별로 보여줄 페이지와 메타데이터를 정의하고, 레이아웃(공통 헤더/푸터)을 적용한다.

### 루트·공통

| 경로 | 역할 |
|------|------|
| `layout.tsx` | 모든 페이지에 공통으로 적용되는 레이아웃. 헤더(로고·NavLinks), 메인 영역, 푸터, GA/AdSense 스크립트, 쿠키 배너를 넣는다. |
| `page.tsx` | 홈(`/`). Hero 문구·CTA·신뢰 배지, 카테고리 퀵 액세스 5칸, Featured 가이드(읽는 시간), 개념 칩, 최신 블로그를 배치한다. |
| `globals.css` | 전역 스타일·CSS 변수(배경/글자/강조색)·prose(본문) 규칙. 테마·타이포의 기준이 된다. |
| `not-found.tsx` | 존재하지 않는 경로 접근 시 404 화면. 홈/블로그로 돌아가기 링크를 준다. |
| `favicon.ico` | 브라우저 탭 아이콘. |
| `robots.ts` | `/robots.txt`를 동적으로 만든다. 크롤러 허용·sitemap URL을 넣는다. |
| `sitemap.ts` | `/sitemap.xml`을 동적으로 만든다. 정적 페이지·블로그/가이드/개념/툴킷/전자책·카테고리·태그 URL을 포함한다. |

### 정적·고정 페이지

**역할:** 법적/운영용 고정 문서. 라우트만 있고 동적 데이터는 없다.

| 경로 | 역할 |
|------|------|
| `about/page.tsx` | 저자·사이트 소개. SectionHeader + 짧은 문단으로 브랜드 신뢰를 보여준다. |
| `privacy/page.tsx` | 개인정보 처리방침. |
| `cookies/page.tsx` | 쿠키 사용·CMP 관련 안내. |
| `terms/page.tsx` | 이용약관. |
| `contact/page.tsx` | 연락처. |
| `disclaimer/page.tsx` | 사이트 고지(정보 제공 목적 등). |

### 콘텐츠 섹션 (목록 + 상세)

**역할:** content/에 있는 MDX를 타입별로 목록으로 보여주고, 슬러그로 상세 글을 렌더링한다. SEO용 메타·JSON-LD도 여기서 채운다.

| 경로 | 역할 |
|------|------|
| `blog/page.tsx` | 블로그 목록. SectionHeader + FiltersBar + ListWithFilters로 검색/필터/정렬 후 ContentCard 목록을 보여준다. |
| `blog/[slug]/page.tsx` | 블로그 글 한 편. 제목·최종 업데이트·본문(MDX), references가 있으면 ReferenceCard, 맨 아래 Disclaimer. BlogPosting·BreadcrumbList JSON-LD. |
| `guides/page.tsx` | 가이드 목록. 동일하게 SectionHeader + FiltersBar + ListWithFilters + ContentCard. |
| `guides/[slug]/page.tsx` | 가이드 한 편. 본문 + 우측 스티키 ToC, 관련 콘텐츠, Reference·Disclaimer. Article JSON-LD. |
| `concepts/page.tsx` | 개념 사전 목록. SectionHeader + ContentCard만 (필터 없음). |
| `concepts/[slug]/page.tsx` | 개념 한 항목. 정의·설명(MDX), 이 개념이 쓰인 글 목록, Reference·Disclaimer. DefinedTerm JSON-LD. |
| `toolkit/page.tsx` | 툴킷 목록. 가이드/블로그와 같은 필터·정렬 구조. |
| `toolkit/[slug]/page.tsx` | 툴킷 한 편. 본문 + Reference·Disclaimer. Article JSON-LD. |
| `books/page.tsx` | 전자책 목록. 출간 목록 톤으로, 상단 안내 문구 + 카드(표지·한 줄 설명·대상 배지·서점 “서점에서 보기” 링크). |
| `books/[slug]/page.tsx` | 전자책 한 권. 표지(있을 때)·제목·대상·설명·본문(MDX)·서점 링크(OutboundLink)·Disclaimer. Book JSON-LD. |

### 아카이브

**역할:** 카테고리/태그로 묶인 통합 목록. blog + guides + toolkit만 포함한다.

| 경로 | 역할 |
|------|------|
| `c/[category]/page.tsx` | 해당 카테고리가 붙은 글만 모아 목록. ContentCard 스타일로 타입 라벨·제목·날짜·요약 표시. |
| `t/[tag]/page.tsx` | 해당 태그가 붙은 글만 모아 목록. 구조는 카테고리 페이지와 동일. |

### 검색·피드·API

**역할:** 사이트 내 검색용 데이터 제공과 RSS 피드 노출.

| 경로 | 역할 |
|------|------|
| `search/page.tsx` | 검색 페이지. SectionHeader + SearchClient(입력창·결과 목록). |
| `search/SearchClient.tsx` | 클라이언트 검색 UI. /search-index.json을 받아 제목·설명·태그로 필터링해 결과를 그린다. trackSearch 호출. |
| `search-index.json/route.ts` | 전체 콘텐츠(blog·guides·concepts·toolkit·books)의 제목·설명·태그·path를 JSON으로 내려준다. |
| `rss.xml/route.ts` | 블로그 글만 모아 RSS 2.0 XML을 반환한다. |

## components/

**역할:** 여러 페이지/레이아웃에서 쓰는 UI 조각. “어떤 데이터를 보여줄지”는 부모가 정하고, “어떻게 그릴지”를 컴포넌트가 담당한다.

| 파일 | 역할 |
|------|------|
| `NavLinks.tsx` | 헤더용 네비 링크 목록. usePathname으로 현재 경로를 보고, 해당 섹션에만 하단 2px 포인트색 인디케이터를 준다. (Client) |
| `SectionHeader.tsx` | 목록 페이지 상단 블록. 배지(선택)·제목·한 줄 설명을 같은 패턴으로 쓴다. |
| `ContentCard.tsx` | 타입(가이드/블로그/개념/툴킷/전자책)별로 좌측 스트립·배경을 다르게 한 카드. 제목·요약·날짜·읽는 시간·카테고리/태그·대상 표시. |
| `FiltersBar.tsx` | 목록 상단 필터 바. 검색 input, 학년/카테고리/태그 select, 최신순·읽는 시간 정렬 버튼. 값은 부모 state로 제어된다. (Client) |
| `ListWithFilters.tsx` | FiltersBar + 필터/정렬된 목록을 한 덩어리로. 검색·필터·정렬 state를 갖고, 필터 결과만 ContentCard로 렌더한다. (Client) |
| `Callout.tsx` | MDX 안에서 쓰는 “핵심 요약” 박스. 좌측 강조선 + 제목(기본 “핵심”) + 본문. |
| `ReferenceCard.tsx` | frontmatter의 references 배열을 받아 “참고 문헌” 제목 아래 링크 카드 목록으로 그린다. |
| `Disclaimer.tsx` | 모든 글 하단에 공통으로 붙는 짧은 고지 문구(정보 제공 목적, 상담 대체 아님 등). |
| `TableOfContents.tsx` | heading 목록을 받아 목차 링크 목록을 그린다. 가이드 상세에서 우측 스티키로 쓴다. (lg 이상에서만 표시) |
| `ConceptLink.tsx` | /concepts/[slug]로 가는 링크. 클릭 시 concept_click 이벤트를 GA4로 보낸다. (Client) |
| `OutboundLink.tsx` | 외부 URL 링크. 클릭 시 outbound_click 이벤트를 GA4로 보낸다. 전자책 서점 링크 등에 사용. (Client) |
| `CookieConsentBanner.tsx` | 쿠키 동의 여부가 없을 때 하단에 배너를 띄우고, “필수만”/“모두 허용” 선택에 따라 쿠키를 저장한다. (Client) |
| `ScrollDepthTracker.tsx` | 스크롤 구간(가이드 25/50/75/90%, 그 외 50/90%) 도달 시 GA4 scroll_depth 이벤트를 보낸다. (Client) |
| `AdSlot.tsx` | AdSense 광고 슬롯을 넣을 때 사용. (Client) |

## content/

**역할:** 실제 글·가이드·개념·툴킷·전자책 설명의 **원본**을 두는 곳. 파일명이 URL 슬러그가 되고, lib/content.ts가 이 폴더를 읽어 목록/상세를 만든다.

| 폴더 | 역할 |
|------|------|
| `blog/` | 블로그 글. 날짜·카테고리·태그가 있으면 목록 정렬·아카이브·필터에 쓰인다. |
| `guides/` | 핵심 가이드(에버그린). 긴 문서에 ToC·관련 콘텐츠가 붙는다. |
| `concepts/` | 개념 사전. 짧은 정의·설명 + “이 개념이 포함된 글” 목록. |
| `toolkit/` | 툴킷(체크리스트·루틴·템플릿). |
| `books/` | 전자책 한 권씩. audience·stores(서점 URL)를 frontmatter에 두면 목록/상세에서 표시한다. |

| 파일 | 역할 |
|------|------|
| `README.md` | MDX frontmatter 규칙 정리. title, description, date, updated, category, tags, grade, ogImage, references 등 필드 설명과 예시. |

## lib/

**역할:** UI가 아닌 **로직**과 **설정**. “어떤 데이터를 가져올지”, “URL/읽는 시간/관련글을 어떻게 계산할지”, “사이트 정보·이벤트 전송을 어떻게 할지”를 제공한다.

| 파일 | 역할 |
|------|------|
| `content.ts` | content/ 폴더를 읽어 슬러그 목록·한 건 조회·타입별 목록·카테고리/태그 목록·아카이브(카테고리/태그별)·날짜(getDatePublished, getDateModified)를 제공한다. gray-matter로 frontmatter 파싱. |
| `content-path.ts` | 타입과 슬러그로 경로 문자열을 만든다. pathForType(“blog”) → “/blog”, fullPath(“guides”, “math-anxiety”) → “/guides/math-anxiety”. |
| `related.ts` | getRelatedByTags: 동일 태그를 가진 다른 글(가이드 기준 관련 가이드/블로그/툴킷 등). getContentReferringToConcept: 해당 개념 슬러그를 태그로 가진 글 목록. |
| `reading-time.ts` | 본문 문자열 글자 수로 “읽는 시간(분)”을 계산한다. 목록·카드에 “N분 읽기”로 쓴다. |
| `headings.ts` | MDX 소스에서 ##, ### 줄을 파싱해 제목 텍스트와 id(슬러그) 목록을 만든다. ToC 렌더·h2/h3 id 부여에 사용. slugify는 제목 → id 문자열 변환. |
| `mdx-components.tsx` | MDX를 React로 렌더할 때 쓰는 컴포넌트 맵. Callout, ConceptLink, h1~h3(id 부여), p, ul, ol, blockquote, code, pre, a 스타일을 통일한다. |
| `site.ts` | 사이트 이름·설명·URL, 저자 이름·URL. 메타·JSON-LD·푸터 등에서 참조한다. |
| `analytics.ts` | GA4 이벤트 전송 함수. trackOutboundClick, trackScrollDepth, trackSearch, trackConceptClick, trackGuideCtaClick. gtag 호출. |
| `posts.ts` | 예전 블로그용 유틸. 필요 시 참고용으로만 사용. |

## public/

**역할:** 빌드 결과에 그대로 복사되어 `/파일명`으로 제공되는 정적 파일. 이미지·폰트·ads.txt 등이 여기 들어간다.

| 파일 | 역할 |
|------|------|
| `ads.txt` | AdSense 도메인 소유 검증. 배포 후 https://도메인/ads.txt 접근 가능해야 한다. |
| `*.svg` | next.svg, vercel.svg 등 기본 제공 아이콘/에셋. 필요에 따라 교체·삭제 가능. |

## 빌드/설정에서 제외

- `node_modules/` — npm 의존성. 버전 관리·문서 대상 아님.
- `.next/` — Next.js 빌드·캐시. 재생성 가능.
- `.env`, `.env.local` — 환경 변수(GA ID, AdSense ID, SITE_URL 등). 저장소에 올리지 않는다.
