# MDX 문법 정리 — 이 웹앱에서 표현되는 것

이 문서는 **현재 웹앱에서 실제로 렌더링되는** MDX/마크다운 문법을 정리한 것입니다. 콘텐츠 타입(블로그, 가이드, 개념, 툴킷, 전자책) 공통으로 적용됩니다.

---

## 1. Frontmatter(YAML)

파일 최상단 `---` ... `---` 안에 메타데이터를 씁니다. SEO·목록·필터에 사용됩니다.

```yaml
---
title: "글 제목"
description: "1~2문장 요약"
date: "2026-03-06"
updated: "2026-03-06"
category: "수포자"
tags: ["작업기억", "인지부하"]
---
```

자세한 필드는 [content/README.md](content/README.md)를 참고하세요.

---

## 2. 제목(Heading)

| 문법 | 렌더링 | 비고 |
|------|--------|------|
| `# 제목` | `<h1>` | 큰 제목, 상단 여백 |
| `## 제목` | `<h2>` | **자동으로 `id` 부여** (제목 텍스트를 슬러그화). 목차·앵커 링크에 사용 가능 |
| `### 제목` | `<h3>` | **자동으로 `id` 부여** |
| `####`, `#####`, `######` | `<h4>` ~ `<h6>` | 기본 HTML 스타일(prose 적용) |

예: `## 왜 수학만 불안할까?` → `<h2 id="왜-수학만-불안할까">` 로 렌더되며, `#왜-수학만-불안할까` 로 앵커 링크 가능.

## 3. 문단·줄바꿈

| 문법 | 렌더링 |
|------|--------|
| 빈 줄로 구분된 텍스트 | `<p>` 단락. 줄 간격·여백이 스타일로 적용됨 |
| 줄 끝 스페이스 두 개 + 엔터 | 줄바꿈 (일부 환경에서만 동작) |

---

## 4. 강조

| 문법 | 렌더링 |
|------|--------|
| `**굵게**` 또는 `__굵게__` | `<strong>` 굵은 글씨 |
| `*기울임*` 또는 `_기울임_` | `<em>` 기울임 |

예: `**작업기억**과 **인지부하**` → 굵은 글씨로 표시.

---

## 5. 목록

| 문법 | 렌더링 |
|------|--------|
| `- 항목` 또는 `* 항목` | `<ul><li>` 번호 없는 목록(들여쓰기·불릿) |
| `1. 항목` | `<ol><li>` 번호 있는 목록 |

중첩 목록은 하위 항목을 스페이스로 들여쓰기합니다.

```markdown
- 상위
  - 하위
```

## 6. 링크

| 문법 | 렌더링 |
|------|--------|
| `[보이는 글자](https://example.com)` | `<a href="...">` 외부 링크. 악센트 색·밑줄 스타일 적용 |
| `[글 제목](/blog/other-post)` | 사이트 내부 링크 (같은 스타일) |

## 7. 이미지

| 문법 | 렌더링 |
|------|--------|
| `![대체 텍스트](이미지경로 또는 URL)` | `<img src="..." alt="...">` |

이미지 경로는 사이트 기준(예: `/og/xxx.png`) 또는 절대 URL을 씁니다. 스타일은 `.prose img` 등 전역 스타일에 따릅니다.

## 9. 코드

| 문법 | 렌더링 |
|------|--------|
| `` `인라인 코드` `` | `<code>` — 배경·테두리·고정폭 글꼴 |
| ` ``` ` 로 감싼 블록 | `<pre><code>` — 코드 블록. 가로 스크롤·패딩 적용 |

코드 블록 바로 다음 줄에 언어 이름을 붙이면 문법 강조(하이라이트)가 동작할 수 있으나, **현재 앱에서는 별도 설정이 없으면 강조 없이** 그대로 표시됩니다.

````markdown
```js
const x = 1;
```
````

## 11. 커스텀 컴포넌트(JSX)

MDX이므로 JSX로 아래 컴포넌트를 직접 사용할 수 있습니다.

### Callout(핵심 요약 박스)

문단 안에서 “핵심만 짧게 강조”할 때 사용합니다.

```mdx
<Callout title="핵심">
  수학 포기는 능력 문제가 아니라 인지부하와 실패 경험의 누적일 수 있습니다.
</Callout>
```

- `title`: 선택. 기본값은 `"핵심"`. 박스 상단에 작게 표시됩니다.
- `children`: 요약 문장(또는 짧은 문단).

---

### ConceptLink(개념 사전 링크)

개념 사전(`/concepts/[slug]`)으로 넘어가는 링크입니다. 클릭 시 GA4 `concept_click` 이벤트가 전송됩니다.

```mdx
<ConceptLink slug="working-memory">작업기억</ConceptLink>
```

- `slug`: 개념 페이지 슬러그(파일명 기준). 예: `working-memory` → `/concepts/working-memory`
- `children`: 선택. 없으면 `slug`가 링크 텍스트로 쓰입니다.

예: `이론은 <ConceptLink slug="인지부하">인지부하</ConceptLink>로 설명할 수 있습니다.`

---

## 12. 표현되지 않거나 제한되는 것

- **MDX에서 export/import**: 문법상 가능하나, 이 앱에서는 frontmatter를 YAML로만 파싱하므로 **export한 값은 레이아웃에서 사용하지 않습니다.** 메타데이터는 frontmatter에만 두는 것을 권장합니다.
- **임의의 React 컴포넌트**: `lib/mdx-components.tsx`에 등록된 컴포넌트만 기본 제공됩니다. 다른 컴포넌트를 쓰려면 해당 파일에 추가해야 합니다.
- **HTML 태그 직접 사용**: MDX는 HTML을 허용하지만, 스타일이 prose와 어긋날 수 있으므로 **가능한 한 마크다운·Callout·ConceptLink로 작성**하는 것이 좋습니다.

## 14. 더 읽어보기 / 관련 콘텐츠 (앱에서 자동 생성)

본문 하단에 나오는 **「더 읽어보기」 → 「관련 콘텐츠」** 블록(예: *[관련 블로그] 왜 아이가 수학을 포기하는가*)은 **MDX에 적는 것이 아니라**, 페이지를 렌더링할 때 **앱이 자동으로 만들어 붙입니다.**

### 어디에 나오나

- **가이드 상세 페이지** (`/guides/[slug]`) 에만 이 블록이 표시됩니다.  
  (블로그·개념·툴킷·전자책 상세 페이지에는 현재 구현되어 있지 않습니다.)

### 어떤 원리로 만들어지나

1. **현재 글의 frontmatter `tags` 사용**  
   보고 있는 가이드의 `tags`(예: `["작업기억", "인지부하", "자기효능감"]`)를 읽어옵니다.

2. **태그가 겹치는 다른 콘텐츠 검색**  
   `lib/related.ts`의 `getRelatedByTags()`가 다음을 합니다.
   - **대상 콘텐츠**: `getAllForArchive()`로 가져온 **블로그·가이드·툴킷** 전체 목록
   - **조건**: 위 목록 중 **현재 글과 태그가 하나라도 겹치는** 글만 필터
   - **제외**: 지금 보고 있는 글(같은 타입 + 같은 slug)은 제외
   - **개수**: 최대 6개까지

3. **타입별 라벨로 표시**  
   각 항목을 타입에 따라 `[관련 블로그]`, `[관련 가이드]`, `[관련 툴킷]` 등으로 붙여서 링크 목록으로 렌더합니다.

따라서 **「[관련 블로그] 왜 아이가 수학을 포기하는가」**가 뜨는 이유는, 그 블로그 글의 frontmatter `tags`에 현재 가이드와 **공통으로 들어 있는 태그**가 있기 때문입니다.  
태그를 잘 넣어 두면, 가이드·블로그·툴킷이 서로 「더 읽어보기」로 자연스럽게 연결됩니다.
