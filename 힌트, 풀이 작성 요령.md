# 수식 작성 요령

에이전트·편집자는 아래 두 갈래를 함께 이해하는 것이 좋다. **(1)** Word에서 옮긴 직후 Pandoc·`docx2mdx`가 하는 자동 치환, **(2)** 그 결과를 `src/content/problems/*.mdx`에 넣은 뒤 Astro 빌드가 하는 치환.

이 프로젝트의 문제 본문(`src/content/problems/*.mdx`)은 **Markdown + 수식(remark-math) + KaTeX(rehype-katex)** 로 렌더된다. Astro MDX 통합과 동일한 파이프라인이며, 문제 지문·힌트·풀이를 HTML로 그릴 때도 같은 방식으로 수식이 처리된다.

## 인라인 · 디스플레이 수식

- **인라인 수식**: 한 줄 안에 붙여 쓸 때는 달러 한 쌍으로 감싼다.

  예: `$a_{n} = a_{1} + (n-1)d$`

- **디스플레이 수식**(한가운데 세로로 크게): 줄바꿈을 넣고 달러 두 쌍으로 감싼다.

  예:

  ```md
  $$
  g(x) = (x^{2} + 3) f(x)
  $$
  ```

- `$` 와 `$` **사이에 불필요한 공백을 넣지 않는다.** (워드 변환 결과처럼 `$f(1) $ $= 2$` 처럼 쪼개지면 수식이 깨지거나 인식이 어긋날 수 있다.) 한 덩어리 수식은 **연속된 한 쌍의 `$` 안**에 넣는다.

## 사용 엔진과 안전 설정

- 수식은 **KaTeX** 로 렌더된다.
- 잘못된 수식이 있어도 빌드가 멈추지 않도록 되어 있고, 오류가 난 부분은 빨간색 등으로 표시될 수 있다. 배포 전에 브라우저에서 해당 문제 페이지를 한 번 확인하는 것이 좋다.

## 분수 `\frac` 자동 처리

- 본문 소스에 `\frac` 이 있으면 빌드 과정에서 **`dfrac`(통크기 분수)** 로 맞추는 처리가 들어간다. 일반적으로 `\frac` 그대로 써도 된다.
- 지수에 사용되는 분수는 `\tfrac`으로 처리한다.

## 문제 페이지에서의 구조(작성자가 지켜야 할 제목)

- **`## 문제`**: 지문(과 필요 시 그림). 다음 `## ...` 전까지가 “문제” 영역으로 잘린다.
- **`## 힌트`**: 단계별로 여러 번 쓸 수 있다. 웹앱은 `## 힌트` 개수만큼 힌트 단계로 나눈다.
- **`## 풀이`**: 모범 풀이 한 블록.

문제 영역에서 **한 줄에 `① … ② … ⑤ …`** 형태로 적으면, 서버 쪽에서 선지를 나누어 객관식 UI에 넣는다. 선지 줄은 관례적으로 문제 블록 **맨 아래**에 둔다.

## DOCX → MDX 변환 도구 (`docx2mdx`)

레포지토리 안에 **Streamlit 기반 변환기**가 있다.

- **경로**: `howlearn-astro/docx2mdx/` (`app.py`, `converter.py`, `postprocess_mdx_pipe.py`, `preview_html.py`)
- **실행 예**: 프로젝트 루트에서 `pip install -r docx2mdx/requirements.txt` 후 `streamlit run docx2mdx/app.py` (또는 `run-docx2mdx.bat`)
- **전제**: 시스템에 **[Pandoc](https://pandoc.org/)** 이 설치되어 **PATH**에서 실행 가능해야 한다. 없으면 변환 단계에서 실패한다.

### Pandoc 호출 방식 (개념)

`converter.docx_to_markdown_pandoc`은 대략 다음에 해당한다.

- 입력: `.docx`
- 출력 형식: `markdown+tex_math_dollars+pipe_tables+raw_html` — 수식은 **`$` / `$$`** 구분자를 쓰는 Markdown으로 맞춘다.
- `--wrap=none` — 불필요한 줄 바꿈 삽입을 줄인다.
- `--markdown-headings=atx` — 제목을 `#` 스타일(ATX)로 내보낸다.
- 이미지 추출 시: `--extract-media <폴더>` 로 미디어 파일을 뽑은 뒤, 스크립트가 경로를 `/images/...` 형태로 다시 쓰고 **WebP** 로 변환한다.

표준 입력으로 Markdown만 후처리할 때는 `postprocess_mdx_pipe.py`가 `converter.postprocess_for_webapp`과 같은 로직을 stdin/stdout으로 적용한다.

### 변환 직후 적용되는 자동 후처리 (`postprocess_for_webapp`)

Word에서 넘긴 원문에 대해 **문자열 단계**에서 아래가 순서대로 적용된다. “변환만 했는데 소스가 이렇게 바뀌었다”는 질문은 여기서 답이 나온다.

1. **줄바꿈 정규화**: `\r\n` 등을 `\n`으로 통일하고 앞뒤 공백을 정리한다.
2. **LaTeX 대괄호 수식**: `\[ … \]` → 디스플레이 **`$$` … `$$`**. 단, 안쪽이 **`[3점]`** 같은 **배점만 표시된 경우**면 디스플레이 수식으로 바꾸지 않고 **그대로 둔다**.
3. **LaTeX 소괄호 수식**: `\( … \)` → 인라인 **`$` … `$`**.
4. **수식 안 `\,` 제거**: Pandoc이 Word 쪽 **띄어쓰기를 `\,`(얇은 공백 명령)** 으로 넣는 경우가 있는데, KaTeX와 충돌할 수 있어 **`$` / `$$` 안의 `\,`는 모두 일반 공백 한 칸**으로 바꾼다.
5. **극한·분수 정리**
   - `\lim_{` → `\lim\limits_{` 로 통일하는 등 KaTeX에서 극한 밑첨자가 잘 보이게 만든다.
   - 밑첨자 안의 `h → 0` 계열 표기를 여러 형태에서 `\rightarrow` 스타일로 맞춘다.
   - `\lim\limits_{…}` 바로 뒤에 오는 `\frac` / `\tfrac` / `\dfrac` 앞에 공백을 넣는 등, 극한 뒤 분수가 붙을 때의 레이아웃을 보정한다.
6. **이항계수 `\binom{n}{r}`** → **`{}_{n}C_{r}`** (좌첨자 조합 기호)로 **통째로 치환**한다. 웹 본문 표기를 통일하려는 의도이므로, **반드시 `\binom{…}{…}` 형태를 유지해야 하면** 변환 결과를 손으로 되돌린다.
7. **큰 연산자가 들어 있는 인라인 `$…$`**: `\lim`, `\sum`, `\int`, 일부 그리스/기호(Σ, ∫ 등)가 포함되면 앞에 **`\displaystyle`** 를 붙인다 (이미 붙어 있으면 생략).
8. **`$…$\`** 패턴: 수식만 있는 줄 끝에 Pandoc이 넣은 **`\\` (hard break)** 를 제거한다.
9. **연속 `$$` 블록 분리**: 특정 패턴에서 디스플레이 수식 블록이 연달아 붙어 remark가 한 블록으로 오인하지 않도록, 사이에 **` $ `** 같은 구분이 끼워 들어가기도 한다. 소스가 지저분해 보이면 **수식 의미가 유지되는 한** 한 줄의 `$$ … $$`로 정리해도 된다.
10. **각도 괄호**: `\langle`, `\rangle` → **`<`**, **`>`** 문자로 바꾼다. 내적·순서쌍을 LaTeX 각도 괄호로 쓰던 경우 표기가 바뀌므로, 필요하면 `\langle` 표기로 되돌린다.
11. **이미지**: `![](…){width=… height=…}` 형태의 **크기 속성 블록**을 제거한다 (레이아웃은 CSS에 맡긴다).

### 이미지 추출을 켠 경우

- Pandoc이 추출한 파일은 **WebP**(품질 85 등)로 바꾸고, 마크다운의 경로는 예를 들어 **`/images/docx2mdx/<이름-stem>-<날짜시간>/media/...`** 처럼 `public/` 기준 URL로 맞춘다.
- Streamlit 화면에 실제 저장 디렉터리와 URL 접두가 표시된다.

### Astro 빌드 단계 (변환기와는 별도)

`src/content/**/*.mdx`는 Vite 플러그인 등으로 **소스에 `\frac` → `\dfrac`** 정규화가 들어가고, **한 줄 `① … ⑤`** 선지는 `<McqChoices>` 계열로 바뀔 수 있다. 즉 **DOCX 도구 출력을 한 번 더 손보는 셈**이다.

### DOCX → 붙여넣기 후 에이전트 체크리스트

1. **`## 문제` / `## 힌트` / `## 풀이`** 같은 구역 제목은 Word에 없을 수 있으므로 **수동으로** 넣었는지 확인한다.
2. **`$` 가 중간에서 끊긴 줄** (`$x$ $+$ $y$` 등)이 없는지 본다. 있으면 **한 수식마다 `$ … $` 한 쌍**으로 합친다.
3. `\binom` 이 `{}_{n}C_{r}` 로 바뀐 것이 문제 의도와 맞는지 본다.
4. `<`, `>` 로 바뀐 각도 괄호가 수학적으로 맞는지 본다.
5. 극한·적분식이 어색하면 `\displaystyle` 과 `\lim\limits` 유무를 확인한다.
6. **Streamlit 미리보기**(`preview_html.py`)는 KaTeX auto-render와 비슷한 그리드로 ⑤지선다를 흉내 낼 뿐이며, **실제 문제 페이지와 픽셀 단위로 동일하지 않을 수 있다.** 최종 검증은 빌드 후 해당 문제 URL에서 한다.
