## 3. 결론

### 3.1 누락 없이 반영된 부분

- **코어 웹 바이탈 3가지(LCP, INP, CLS)** 및 **FCP, Speed Index, TBT** 전부 계획에 포함되어 있음.
- 보고서에 나온 **최상 경로 지연(873ms)** 은 3.2(Preconnect), 3.8(서버/인프라)에서 직접 다룸.
- Lighthouse가 다루는 **기회·진단** 항목 대부분이 계획의 섹션 2와 3에 1:1 또는 통합 형태로 대응됨.
- **폰트, preconnect, 스크립트, 이미지, CLS, 번들, 캐시, passive listener**까지 계획에 명시되어 있음.

### 3.2 선택·간접 반영으로 충분한 항목

- **Minify CSS/JS**: Next 빌드 기본 동작 + 3.6 번들 분석으로 점검 가능 → 별도 계획 항목 불필요.
- **Text compression**: 3.8 인프라/CDN에서 처리 → 계획 범위 내.
- **Preload key requests**: LCP는 이미 `priority`로 처리됨. 추가 preload는 선택 사항.
- **document.write / non-composited animations**: 사용 시 3.6·3.9 또는 코드 리뷰로 커버 가능.

### 3.3 최종 판단

**코어 웹 바이탈과 PageSpeed/Lighthouse에서 지적하는 성능 관련 문제는 계획에 누락 없이 반영되어 있다.**  
선택·간접 반영 항목만 보강하고 싶다면, 3.8에 “텍스트 압축(Gzip/Brotli) 확인”, 3.6에 “document.write 미사용 확인” 한 줄을 추가해도 되지만, 필수는 아님.
