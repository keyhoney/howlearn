# Cursor AI 개발 지침서

아래는 **Next.js(App Router) + MDX** 기반 블로그를 **Cursor AI**에게 구현을 요청하기 위한 지침 예시입니다. 주제는 “왜 아이가 수학을 포기하는가”로, **학부모 대상 교육/심리 콘텐츠** 블로그를 상정합니다. 각 항목은 필요한 기능과 설정을 설명하며, 참고 가능한 공식 문서도 함께 제시합니다.

## 1. 프로젝트 기본 설정  
1. **Next.js 프로젝트 생성**: 최신 안정 버전(예: Next.js 16 이상)으로 프로젝트 생성(`create-next-app`). App Router 사용을 명시합니다.  
2. **Tailwind CSS 설정**: 공식 가이드에 따라 Tailwind를 설치 및 구성합니다(프로젝트에 `tailwind.config.js` 작성). 필요 시 [Tailwind Typography](https://tailwindcss.com/docs/typography-plugin) 플러그인도 추가합니다.  
3. **MDX 플러그인 설치**: MDX를 사용하려면 `@next/mdx` 등 MDX 관련 패키지를 설치합니다. 공식 문서에 따르면 Next.js는 프로젝트 내부의 `.mdx` 파일을 페이지나 컴포넌트로 처리할 수 있으며, MDX를 통해 동적 React 컴포넌트도 포함할 수 있습니다【3†L509-L517】.  
4. **next.config.mjs 구성**: `next.config.mjs`에서 `withMDX`를 적용하여 MDX를 빌드에 포함합니다(`pageExtensions`에 `.mdx` 추가). 또한 환경변수나 리다이렉트 같은 기본 설정도 이 파일에 둡니다.  
5. **`mdx-components.tsx` 추가**: App Router에서 MDX를 사용하려면 프로젝트 루트(예: `src/`)에 `mdx-components.tsx` 파일을 만들어야 합니다. 이 파일에서 MDX 내장 컴포넌트(`h1`, `p` 등)를 커스터마이징할 수 있습니다【3†L593-L600】.  

## 2. 콘텐츠 폴더 구조 및 라우팅  
- **콘텐츠 저장소**: 예를 들어 `content/` 폴더를 만들어 학습 관련 MDX 파일(예: `왜-수학을-포기하는가.mdx`)을 저장합니다.  
- **동적 라우트 페이지**: `app/blog/[slug]/page.tsx` 파일을 만들어 MDX 파일을 불러옵니다. Next.js 공식 예시처럼 `generateStaticParams` 함수를 사용해 prerender 대상 슬러그를 정의하고, 각 경로마다 `import(\`@/content/${slug}.mdx\`)`로 포스트 컴포넌트를 렌더링합니다【22†L696-L704】【22†L707-L710】. 예를 들면:

  ```tsx
  // app/blog/[slug]/page.tsx
  export default async function Page({ params }: { params: { slug: string } }) {
    const { slug } = params;
    const { default: Post } = await import(`@/content/${slug}.mdx`);
    return <Post />;
  }
  export function generateStaticParams() {
    return [{ slug: 'welcome' }, { slug: 'about' }]; // 실제 슬러그 목록으로 교체
  }
  ```
  이처럼 동적 라우팅과 `generateStaticParams`를 사용하면 MDX 기반 블로그 글이 정적으로 생성됩니다【22†L696-L704】【22†L707-L710】.  
- **추가 페이지**: `app/about/page.tsx`, `app/privacy/page.tsx` 등 고정 페이지를 만들어 저자 소개와 개인정보처리방침 등을 작성합니다. (특히 EU/한국 법규 준수를 위해 쿠키, 개인정보 방침 페이지는 필수입니다.)  

## 3. SEO 및 애드센스 설정  
- **메타데이터**: Next.js의 [Metadata API](https://nextjs.org/docs/app/api-reference/file-conventions/metadata)를 활용해 `<title>`, `<meta>` 태그를 각 페이지에 설정합니다. 블로그 글에는 `<title>`, `<description>`, OG 이미지 URL 등을 포함하고, `canonical` 링크도 삽입합니다. (예: `generateMetadata` 함수 이용)  
- **robots.txt**: `app/robots.txt` 파일을 만들어 모든 크롤러가 사이트를 인덱싱하도록 허용하고, 사이트맵 경로를 지정합니다. 예시:  
  ```
  User-Agent: *
  Allow: /
  Sitemap: https://yourdomain.com/sitemap.xml
  ```  
  Next.js 공식 문서에서도 앱 루트에 `robots.txt`를 두어 크롤러 접근을 제어하고 사이트맵을 안내하라고 권장합니다【16†L481-L489】.  
- **sitemap.xml**: 사이트맵은 별도 라우트(`app/sitemap.ts`)나 정적 파일로 구현합니다. 블로그 글, 카테고리, 태그 페이지 등을 포함하도록 분할하면 검색엔진 크롤링 효율이 높아집니다. (예: `/sitemap.xml`에 여러 개의 세부 sitemap 경로를 연결)  
- **ads.txt**: 애드센스 도메인 인증을 위해 `public/ads.txt` 파일을 프로젝트 루트의 `public` 폴더에 추가합니다. GitHub Discussions에서도 언급하듯, 배포 후 반드시 `https://yourdomain.com/ads.txt` 로 접근 가능한지 확인해야 합니다【11†L327-L330】.  
- **CMP 도입**: 유럽(EEA/영국) 사용자 대상 개인화 광고를 위해서는 IAB TCF 인증 CMP가 필요합니다. Google 공식 가이드에 따르면 2024년부터 EEA/영국 사용자에게 맞춤 광고를 내보낼 때 TCF 연동된 인증 CMP 사용이 필수입니다【14†L46-L54】. (스위스는 2024년 7월부터 해당) 따라서 초기에 CookieConsent 라이브러리나 외부 CMP를 적용할 준비를 합니다.  

## 4. 배포 및 최종 검토  
- **Vercel 배포**: Vercel에 리포지토리를 연결하고 배포합니다. Tailwind와 Next.js가 잘 빌드되는지 확인합니다. 배포 후 `robots.txt`, `sitemap.xml`, `ads.txt` 같은 정적 파일 경로가 올바르게 노출되는지 검증합니다.  
- **애널리틱스**: Google Analytics(페이지뷰, 스크롤 이벤트)를 도입합니다. Cursor AI로 코딩할 때 `@next/third-party`나 `next/script`를 사용해 GA4와 애드센스 스크립트를 불러오면 권장됩니다.  
- **체크리스트 점검**: 개인정보처리방침, 쿠키 동의 배너, 저자 소개 페이지 유무를 확인합니다. 특히 심리·교육 정보 콘텐츠이므로 치료법 안내나 진단과 혼동되지 않도록 고지 페이지(`disclaimer`)를 둡니다.  

## 5. Cursor AI 프롬프트 예시  
다음은 Cursor AI에게 전달할 **프롬프트 샘플**입니다. 실제 지침은 이 예시를 참고해 세부 요구사항에 맞게 조정하세요.

```
Next.js 프로젝트를 세팅해줘. 프레임워크는 Next.js(App Router), 스타일은 Tailwind CSS. 
1. `package.json`에 MDX 관련 의존성을 추가하고, `next.config.mjs`로 `@next/mdx`를 설정해줘【3†L509-L517】.
2. `app/` 디렉터리 구조를 만들고, `content/` 폴더에 MDX 파일(블로그 포스트)을 놓자.
3. `app/blog/[slug]/page.tsx`를 생성해서 동적 라우트로 MDX를 로드하도록 코딩해줘. `generateStaticParams`를 써서 슬러그 목록을 반환하게 하고【22†L696-L704】【22†L707-L710】, 각 페이지에서 import(`@/content/${slug}.mdx`) 하자.
4. `app/about/page.tsx`, `app/privacy/page.tsx` 등 정적 페이지를 추가해 “저자 소개”와 개인정보 처리방침을 작성해.
5. Tailwind 설정을 완료하고, 글 본문에 적당한 스타일을 적용(예: `prose` 클래스).
6. `_app` 또는 상위 레이아웃에서 Google Analytics와 애드센스 스크립트를 추가(광고를 위해 `<Script>` 컴포넌트 사용 권장).
7. `public/ads.txt`를 만들어 게시자 ID 정보를 추가. 배포 후 반드시 `https://도메인/ads.txt`로 접근 가능한지 검증【11†L327-L330】.
8. `app/robots.txt` 파일을 추가하여 모든 크롤러에 허용 설정과 사이트맵 링크를 기재【16†L481-L489】.
9. `app/sitemap.ts` 등을 만들어 사이트맵을 생성. 블로그 글, 카테고리, 태그 페이지를 포함시키자.
10. (유럽 대상) CMP 동의 배너 구현을 잊지 마. Google이 요구하는 TCF 인증 CMP를 사용하도록 준비한다【14†L46-L54】.
11. 마지막으로 빌드해서 `/blog` 목록과 각 포스트가 잘 보이는지, SEO 메타태그(제목, 설명, canonical 등)가 제대로 설정됐는지 확인한다.

위 단계를 Cursor AI에 지시하면 **MDX 기반 블로그 구조**, **SEO 최적화 설정**, **애드센스 요구 사항**이 반영된 코드를 얻을 수 있을 것입니다. 필요한 라이브러리나 구체적 구현 예제는 다음 공식 문서를 참고하세요. MDX 설정 및 라우팅은 Next.js 가이드【22†L696-L704】를, robots.txt 생성은 공식 권장사항【16†L481-L489】을, 애드센스/동의(CMP)는 구글 가이드【14†L46-L54】【11†L327-L330】를 참고했습니다. 
```
